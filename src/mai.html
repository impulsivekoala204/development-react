WEBSITE

Hacker News

For this project, I had to redesign a website based on UX issues that I came across and I decided to pick the YC Hacker News website. This website is an excellent source of rich tech news compiled from multiple different sources but the design is still the same as it was years ago. There are several issues ranging from readibility to hierarchy which is why I decided to pick this website.


PROBLEMS

  • Readibility Issues [Headlines, Functionality below Headline]
  • Poor size and color contrast between different text
  • Logo and Header hierarchy unclear [logo looks the same as text]
  • Element hierarcy unclear
  • Number of upvotes is hard to find and immediately clear
  • Hard to find source
  • The two core CTAs(calls to action) which are Hide and Comment are not clear
  • The search bar is at the bottom of the page

ACCESSIBILITY

From WAIVE, I agreed with the result that the text is too small at several places which makes it hard to read. WAIVE also showed poor heading and element structure which I also agreed with. Lastly WAIVE showed a lot of contrast errors with the colors as well which I thought were valid because of the poor hierarchy which makes the elements very hard to read or even distinguish properly.
